Операторы в C++

Операторы в C++ - это символы, которые выполняют операции над переменными и значениями. Они позволяют производить вычисления, сравнивать значения, управлять потоком выполнения программы и многое другое.

Арифметические операторы

Оператор Описание Пример
+ Сложение a + b
- Вычитание a - b
* Умножение a * b
/ Деление a / b
% Остаток от деления a % b
++ Инкремент a++ или ++a
-- Декремент a-- или --a 
int a = 10, b = 3;
int sum = a + b;      // 13
int diff = a - b;     // 7
int product = a * b;  // 30
int quotient = a / b; // 3 (целочисленное деление)
int remainder = a % b;// 1
a++;                  // a становится 11
--b;                  // b становится 2

Операторы присваивания

Оператор Пример Эквивалент
= a = b a = b
+= a += b a = a + b
-= a -= b a = a - b
*= a *= b a = a * b
/= a /= b a = a / b
%= a %= b a = a % b

Операторы сравнения

Оператор Описание Пример
== Равно a == b
!= Не равно a != b
> Больше a > b
< Меньше a < b
>= Больше или равно a >= b
<= Меньше или равно a <= b

Логические операторы

Оператор Описание Пример
&& Логическое И a && b
|| Логическое ИЛИ a || b
! Логическое НЕ !a 
bool isTrue = true, isFalse = false;
bool result1 = isTrue && isFalse; // false
bool result2 = isTrue || isFalse; // true
bool result3 = !isTrue;           // false

Битовые операторы

Оператор Описание Пример
& Битовое И a & b
| Битовое ИЛИ a | b
^ Битовое исключающее ИЛИ a ^ b
~ Битовое НЕ ~a
<< Сдвиг влево a << b
>> Сдвиг вправо a >> b 
unsigned int a = 5;    // 0101
unsigned int b = 3;    // 0011
unsigned int r1 = a & b; // 0001 (1)
unsigned int r2 = a | b; // 0111 (7)
unsigned int r3 = a ^ b; // 0110 (6)
unsigned int r4 = ~a;    // ...11111010 (зависит от размера int)
unsigned int r5 = a << 1; // 1010 (10)
unsigned int r6 = a >> 1; // 0010 (2)

Условный (тернарный) оператор

Синтаксис: условие ? выражение1 : выражение2

int a = 10, b = 20;
int max = (a > b) ? a : b; // max будет равно 20

Совет по использованию операторов

Используйте скобки для явного указания порядка выполнения операций, даже если он соответствует стандартному приоритету. Это сделает код более читаемым и поможет избежать ошибок.
